projects

Some miscellaneous projects that I've done just for fun.

  • chibi: a distributed SQL database written using only go standard library
  • haystack: a distributed file system with an architecture similar to Facebook's Haystack. This entails a directory that is in this case a Raft cluster for added availability and reliability. The directory maps files to logical volumes which are large append-only files on disk that contain many smaller files. Clients ask the directory where to write files and where to read them, meaning the directory doesn't directly interact with stores. Replication is done using the idea that a single logical volume exists in many store servers and a single file is mapped to a single logical volume meaning a copy of the file is on N store servers.
  • mc: a C compiler written in Go. The project contains a lexer, parser, IR generation and x64 assembly generation.
  • mci: a proof of concept distributed CI/CD pipeline service containing a website interface. There is a main orchestrator which then distributes work amongst agents that execute pipelines defined using yaml. The agents execute pipelines in a given docker image such as alpine.
  • serie: a embeddable time series database that employs a custom LSM tree modified to more efficiently query time series data also employs delta-delta encoding to reduce the amount of memory required for storing the data. Contains a HTTP api and a simple distribution mechanism using consistent hashing. There is also a parser for a custom query language that can be used to aggregate data.
  • dcache: a fast distributed in-memory cache built using go. It uses gRPC to communicate between nodes, serf to manage the cluster, and raft for keeping the distributed consensus.
  • distsql: a distributed SQLite3 database built using go. It has a HTTP api and nodes communicate using gRPC. Distributed consensus is implemented using raft.
  • rc: a lisp compiler written in Rust that compiles code directly into ELF binaries without any dependencies. It parses s-expressions and generates an IR. Small optimizations like constant folding are done and then the IR is translated into x64 ELF binaries.
  • levelsql: a barebones SQL implementation on top of leveldb it encodes rows and tables and stores them as key-value pairs.
  • sagasu: a minimal service discovery tool that has a cluster for storing a service registry; very simple version of consul
  • upfi: a super simple file hosting platform
  • gocry: a proof of concept ransomware
  • iso: a distributed filesystem written in C
  • karu: a bitcask style key-value store for linux written in C++ mainly done to learn what it's like to develop larger projects with C++.
  • stupidhttp: a small http server built in go from scratch without net/http supporting a subset of HTTP/1.1 and HTTPS
  • juoru: a gossip protocol for peer-to-peer communication
  • sch: a small and fast chat application written using Tokio and Rust
  • hashvis: a utility that generates a unique ASCII art visualization from the contents of a given file. It reads the input file, hashes its content to seed a random number generator, and then uses this generator to create a "random walk" pattern on a grid. This pattern is then rendered as ASCII characters to the console, providing a visual fingerprint of the file.
