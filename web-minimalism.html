the modern and bloated web

17-9-2022

I'm probably not the only who who thinks that most modern websites are bloated with horrendous amounts of javascript and other unnecessary resources.

In my opinion the purpose of a web page is to communicate information through a browser. In addition, some web pages need to communicate back and forth with the user. The gist of it is that HTML files are exported from a server to a browser and those HTML files display some kind of content. Nowadays the process of communicating those HTML files is done inefficiently with the use of javascript frameworks and other resources which add hundreds of kilobytes of useless content.

Why lay the burden of loading so much unnecessary data on a reader that might not have a good internet connection? I started writing this post after trying to read the news on a finnish news site and seeing my browser load over 10 megabytes of content for a short news article. Having good internet it's sometimes hard to notice these things, but imagine someone with a 2mb/s internet connection trying to read the news; it's going to take many seconds just loading up the article. If the only thing the user had to load was the HTML content and some ads to support the business, it would still load almost instantly on any internet connection.

Of course, I've worked a lot with the javascript frameworks in the past and I know how easy they make development. I just don't think that easier development should cost the user their time and bandwith. I've taken a liking to just rendering dynamic HTML content on the server and returning the plain HTML file to the user. The ease of development still is apparent, but no burden has been layed on the user. Maybe I'm missing something?
